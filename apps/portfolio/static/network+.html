Networking Concepts Overview

To succeed in the Network+ certification, understanding networking concepts is crucial. The following are key areas to focus on:

1.0 Networking Concepts

1.1 OSI Reference Model

The OSI model is a foundational concept in networking. It is split into seven layers:

  • Layer 1 - Physical
    • The Physical Layer of the OSI represents the bits on the wire or the medium at which these bits travel upon
    • Protocols: IEEE 802.3
  • Layer 2 - Data Link
    • The Data link layer represents communication in a Local Area network and frequently uses MAC addresses to reference machines
    • Protocols: Ethernet, Spanning Tree Protocol (STP)
  • Layer 3 - Network
    • The Network link layer represents communication in that is used in routing mainly to route paket through the internet.
    • Protocols: ICMP, IP, ARP
  • Layer 4 - Transport
    • The Transport link layer represents communication that defines rules for communiation, integrity, reliability, of data
    • Protocols: UDP, TCP, SCTP
  • Layer 5 - Session
    • The session layer mainly handles manages connections and provides authentication
  • Layer 6 - Presentation
    • The presentation layer handles, encryption and decryption
  • Layer 7 - Application
    • The Application layer mainly handles web request, emails things of that nature
    • Protocols: HTTP/HTTPS, SMTP

Understanding these layers is essential as each layer has specific responsibilities and protocols.

1.2 Networking Appliances and Functions

Several networking appliances and services play important roles in networking environments:

  • Router, Switch, Firewall, Load Balancer
    • Router - is a network device that is used to forward IP traffic to the internet
    • Switch - is a network device that is used to forward typically layer 2 traffic using ARP and MAC tables, switches can be layer 3
    • Firewalls - is a network device that controls ingress and outgress traffic to filter out malicous traffic vs legitmate traffic
    • Load Balancer - a network device that is used to maximize device utlization
  • Intrusion Detection Systems (IDS), Intrusion Prevention Systems (IPS)
  • Content Delivery Networks (CDN), VPNs
  • Virtual Private Cloud (VPC) and Network Security Groups

1.3 Cloud Concepts and Connectivity Options

As networking evolves, cloud technologies become integral. These are some critical concepts to understand:

  • Cloud Deployment Models: Public, Private, Hybrid
  • Service Models: SaaS, IaaS, PaaS
  • Connectivity: VPN, Direct Connect
  • Cloud Gateways: Internet Gateway, NAT Gateway

1.4 Networking Ports and Protocols

Understanding common networking ports and protocols is essential for configuring and troubleshooting networks:

  • FTP (Ports 20/21), HTTP (Port 80), HTTPS (Port 443)
  • SSH (Port 22), SMTP (Port 25), DNS (Port 53)
  • Common Traffic Types: Unicast, Multicast, Broadcast

1.5 Transmission Media and Transceivers

Transmission media and the use of transceivers are fundamental in establishing physical network connections. Key technologies include:

  • Wired Media: Ethernet (802.3), Fiber (Single-mode vs. Multimode), Coaxial
  • Wireless: 802.11 (Wi-Fi), Cellular, Satellite
  • Transceivers: SFP, QSFP, LC, SC connectors

1.6 Network Topologies

Various network topologies affect how devices communicate within a network:

  • Mesh, Star, Hybrid, and Point-to-Point
  • Three-Tier Hierarchical Model: Core, Distribution, Access
  • Traffic Flow Types: North-South, East-West

1.7 IPv4 Network Addressing

Knowledge of IPv4 addressing and subnetting is crucial for network design:

  • Subnetting: VLSM, CIDR
  • Class A, B, C, D, E Addresses
  • Private IP Addressing (RFC1918), APIPA

1.8 Evolving Network Environments

As networks evolve, technologies like SDN, SD-WAN, and Zero Trust Architecture (ZTA) are becoming more important:

  • Software-Defined Networking (SDN) and SD-WAN
  • Zero Trust Architecture (ZTA) and SASE/SSE
  • IPv6 Addressing and Compatibility
